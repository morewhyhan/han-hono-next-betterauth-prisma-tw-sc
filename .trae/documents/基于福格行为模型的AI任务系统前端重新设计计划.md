# 基于福格行为模型的AI任务系统前端重新设计计划

## 1. 现有问题分析

1. **设计冗余**：当前页面设计过于复杂，元素过多
2. **首页不合理**：福格三要素显示在首页没有针对性
3. **三要素使用不当**：福格三要素应针对每个任务，而非全局显示
4. **布局混乱**：页面元素布局不够清晰，缺乏层次感
5. **功能重叠**：不同页面功能存在重叠

## 2. 新设计原则

1. **极简设计**：减少冗余元素，突出核心功能
2. **任务中心化**：以任务为核心，福格三要素针对每个任务显示
3. **清晰层次**：合理的页面结构，清晰的视觉层次
4. **功能分离**：每个页面专注于单一功能
5. **直观易用**：简洁的用户界面，易于理解和使用

## 3. 页面结构重新设计

### 3.1 首页（/home）
**核心功能**：展示今日任务概览，突出当前任务
**设计要点**：
- 简化布局，只显示最重要的信息
- 今日任务列表，按时间排序
- 当前执行的任务突出显示
- 快速添加任务入口
- 移除全局福格三要素显示

### 3.2 任务详情页（/tasks/[id]）
**核心功能**：展示单个任务的详细信息，包括福格三要素
**设计要点**：
- 任务基本信息（标题、描述、截止日期等）
- 福格三要素分析（动机、能力、提示）
- AI建议
- 任务操作（开始、完成、延期等）

### 3.3 任务列表页（/tasks）
**核心功能**：展示所有任务，支持筛选和排序
**设计要点**：
- 任务列表，支持按状态、优先级、日期筛选
- 简洁的任务卡片，显示关键信息
- 快速操作入口
- 批量操作支持

### 3.4 AI聊天页（/ai-chat）
**核心功能**：与AI交互，制定任务计划
**设计要点**：
- 简化聊天界面，突出对话内容
- 移除冗余说明
- 清晰的输入区域

### 3.5 日历视图页（/calendar）
**核心功能**：日历化展示任务
**设计要点**：
- 简洁的日历视图
- 任务标记清晰
- 快速添加任务入口

### 3.6 设置页（/settings）
**核心功能**：系统设置和个性设置
**设计要点**：
- 清晰的设置分类
- 简洁的表单设计
- 直观的选项布局

## 4. 组件重新设计

### 4.1 任务卡片组件
**设计要点**：
- 简洁的卡片设计
- 显示任务标题、时间、优先级
- 福格三要素评分（仅在任务详情中显示）
- 快速操作按钮

### 4.2 福格三要素组件
**设计要点**：
- 针对单个任务显示
- 简洁的评分展示
- AI建议和改进方案
- 只在任务详情页使用

### 4.3 AI聊天组件
**设计要点**：
- 简化的聊天界面
- 清晰的消息气泡
- 简洁的输入区域
- 移除冗余装饰

## 5. 实现步骤

1. **更新首页设计**：简化布局，突出今日任务
2. **创建任务详情页**：展示单个任务的福格三要素
3. **优化AI聊天页**：简化界面，突出对话内容
4. **简化日历视图**：去除冗余元素
5. **更新设置页**：清晰的分类布局
6. **优化组件设计**：简化所有组件，减少冗余

## 6. 预期效果

1. **简化直观**：页面设计简洁，易于理解
2. **任务中心化**：以任务为核心，功能明确
3. **合理使用福格模型**：三要素针对每个任务，有实际意义
4. **清晰的导航**：页面结构清晰，导航直观
5. **良好的用户体验**：简洁的界面，流畅的交互

## 7. 技术实现

- 使用现有技术栈：Next.js + Tailwind CSS + Radix UI
- 保持组件化设计，提高复用性
- 优化性能，减少不必要的渲染
- 确保响应式设计，适配不同设备

这个重新设计计划将使系统更加简洁、直观，突出核心功能，合理使用福格行为模型，提供更好的用户体验。
